Buttons

For GCC, we will go by the following rules:

  • General Rules:
    • CAPS: No all CAPS. If needing to emphasize a button's entire text, use the "btn-lg" class.
    • Bold: No bold text, unless needing to emphasize a word or phraze without making entire text of button bold. If needing to emphasize a button's entire text, use the "btn-lg" class.
    • Italics: No italics (confirm)
    • Alignment: Situational, but generally we will use the default "text-start" alignment.
    • Width: Widths will be situational.
  • Main Content
    • Color: Main content buttons will almost always be btn-primary. In some cases, when there are multiple buttons used within the main content area, a mix of btn-primary and btn-secondary can be used.
  • Left & Right Sidebars
    • Color: Main content buttons will almost always be btn-secondary. In some cases, when there are multiple buttons used within the main content area, a mix of btn-secondary and btn-primary can be used.

For detailed information, go to Bootstrap Buttons

Google Sheets

Take careful note of the HTML structure and styles. Keep in mind that, because Google Sheets is NOT responsive, we've applied a slightly taller height than required so that when the screen is sized down there will be no vertical scroll bar caused by the horizontal scroll bar that will be added by the browser. The height of the iframe will always need to be set manually in the HTML. See below for an example of an embedded Google Sheet.

Example google sheet to embed: https://docs.google.com/spreadsheets/d/1zGHMXxZMznAPILhaFz2POmSZTuv7R7Dg1KCCsRjUUx4/edit#gid=0

<div class="overflow-hidden my-3">
  <iframe class="container-fluid p-0 position-relative" style="left:-1px;height:525px;" src="[url]?gid=0&single=true&widget=false&headers=false&chrome=false&range=A:C&gridlines=false"></iframe>
</div>

Parameters:

  • gid=0 - ID of the Google Sheet to display. (The first sheet is almost always 0)
  • single=true - Show single sheet? If no, then set the widget OR chrome parameter to true to show sheet tabs.
  • widget=false - Show sheet tabs? (Tabs will be at the bottom.)
  • headers=false - Show row numbers and column letters?
  • chrome=false - Show sheet title and footer along with tabs? (Tabs will be at the top.)
  • range=A:C - Range to render/display.
  • gridlines=false - Show grid lines?

The Small Image Text Block is intended to replace images which are floated left or right within content -- this allows inline images with corresponding paragraph text to be responsive.

Wide Images

  • Large screens:
    • Images will be automatically sized down to fit within the content using class img-fluid
    • To make images larger or smaller, you will need to adjust the column sizes. Column sizes must add up to 12 -- the image columns below are set to col-lg-4, thus making the paragraph text column col-lg-8 to equal 12.
    • An image will not become larger than its original upload dimensions.
  • Small screens:
    • Images will fit the full width of the page as long as their width is greter than or equal to the width of the page -- resize images prior to uploading to the intended size when viewing on mobile or smaller screens.
    • See below for information regarding tall images.

Tips and Tricks

Non-Breaking Hyphens

Use the non-breaking hyphen (&#8209;) in telephone numbers and date ranges if intending to keep the text together, i.e. in table cells that may resize and make telephone numbers two lines instead of one. This may need to be used in combination with non-breaking spaces (&nbsp;).

Empty Tags

By default, CKEditor does not allow empty tags and will insert the &nbsp; HTML entity. If you are needing to insert an empty div, for instance, without affecting width and height, then use the BOM HTML entity &#xfeff;. However be sure to check for cross-browser compatibility.
